4 总结

4 总结

养成编译前查看帮助文档的好习惯

1）执行如下命令，熟悉编译配置项

./configure -h

2）查看 README 文档

大部分开源库源码目录都有 README 文档，建议在编译前阅读熟悉一遍。

3）查看 INSTALL 文档

部分开源库源码目录有 INSTALL 文档，对编译配置有详细说明，配置前需要阅读一遍。

4）浏览官网

官网有比较详细的开源库介绍，源码下载链接，使用说明等重要信息，并且一般会很全面，非常值得一读。

5）分析 Makefile 或者 CMakeLists.txt

有些交叉编译问题可能比较棘手，需要阅读 Makefile 或者 CMake 文件来分析。

6）浏览 GitHub Issues

我们遇到的问题，大部分别人都已经遇到过，一种方式是通过开源库 GitHub 的 Issues 搜索类似问题的讨论及解决方案。

配置方式

开源库一般有两种构建方式，一种是 configure + Makefile 构建；一种是 CMake 构建；通过 configure + Makefile 构建的开源库用 configure 配置；通过 CMake 构建的开源库用 CMake 配置。

  • configure：传统软件工程构建方式，大部分开源库使用该方式构建工程；
  • CMake：CMake 是相对比较新的工具，使用简单，特别适合跨平台的软件工程构建。建议学习 CMake 工具的使用，我自己的项目开发一般都是使用 CMake 构建；

当然也有部分开源库同时支持以上两种配置方式，可以根据自己的习惯选择一种方式进行配置编译。

尽量不改动源码

如无必要，尽量不要改动源码，包含 Makefile 文件；如果确实有特殊需要，可以通过 configure 做定制化配置编译；最后的选择才是改动 Makefile ，改动源码。

配置编译失败可能原因

  • 缺少依赖的工具；
  • 缺少依赖的库；
  • 依赖工具版本过低或过高；
  • 依赖库版本过低或过高；

交叉编译的开源库多了就会发现其实套路都差不多
